Game Engine Architecture

Entity System, Player Logic, World Map & Save System

With 100% of functions now named (476 total), the complete architecture of Klonoa: Empire of Dreams is mapped. The game is structured around a callback-based state machine with per-frame entity updates, a scene transition system, and a modular level loading pipeline.

Main Game Loop

AgbMain()                          @ 0x08000470 — entry point
  ├─ Clears IWRAM/VRAM/OAM/palette
  ├─ Initializes sound, input, display
  └─ Main loop:
      ├─ ReadKeyInput()               @ debounce + reset combo
      ├─ GameUpdate()                 @ if not paused:
      │   ├─ UpdateWorldMapNodeState()
      │   ├─ UpdatePlayerInput()
      │   ├─ InitPlayerCollision()
      │   └─ UpdateWorldMapCursor()
      ├─ UpdatePaletteAnimations()   @ 0x2C palette slots
      └─ VBlank wait + DMA transfers

Entity System

The game uses a fixed-size entity array at 0x03002920, with each entry being 28 bytes. Entity slot 0 is always the player.

Entity Update Pipeline

EntityBehaviorMasterUpdate()       @ 0x08014760 — 13,237 lines (LARGEST function)
  └─ For each active entity:
      └─ EntityUpdateDispatch()       @ 0x080101E8 — switch on entity type
          ├─ EntityGravityAndFloorCheck()
          ├─ EntityScrollBoundsCheck()
          └─ Type-specific handler...

Player Actions

  • Grab/Throw: PlayerGrabInputCheck detects grab input, EntityStateSwitch_Carried handles carried entity states
  • Collision: CheckTileCollisionVertical and CheckTileCollisionSloped check against the tile collision map
  • Death/Respawn: PlayerRespawnOrDeath saves facing direction and resets scroll counters

Scene Transition System

The game uses a callback-stack system for scene transitions. Each transition is a function that gets installed into the callback chain and removes itself when complete.

Save System (EEPROM)

The game uses EEPROM for saving via a 5-function pipeline with retry logic: 

SaveGameToSRAM()                   @ disable IRQ, stop DMA, write with retry
  └─ SaveGameRetryLoop()           @ up to 10 retries in 64-iteration loop
      └─ EepromWriteSector()       @ build command + 64-bit data, DMA transfer
          └─ EepromDmaTransfer()   @ configure WAITCNT, DMA3 transfer

SaveGameWithVerify()                @ write + read-back verification
  └─ EepromVerifySector()          @ read + compare 4 halfwords
      └─ EepromReadSector()       @ build address command, read 64 bits

LoadGameFromSRAM()                  @ backup DMA regs, read, restore

Level Loading Pipeline

Each of the 9 worlds has 2 vision variants (18 level loaders total) plus a boss arena loader. Each loader DMA-copies ROM tile/map/palette data into VRAM for that specific level.
