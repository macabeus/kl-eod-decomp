Program Flow

This diagram shows the complete execution flow of Klonoa: Empire of Dreams, from the AgbMain entry point through the main game loop, scene dispatch, and transition system.

The game uses a callback-driven state machine: a central callback block at 0x03003510 holds function pointers that define the current scene's behavior. Scene transitions work by fading out over 16 frames, freeing resources, installing new callback pointers, and fading back in.

Each frame, a VBlank interrupt fires to handle rendering (OAM DMA, BG scroll, palette updates), then the main loop reads input, updates timers, and dispatches to the current scene's update function. When a transition is triggered (gControlBlock[0] < 0), the FadeOutController takes over until the fade completes and the new scene is loaded.
